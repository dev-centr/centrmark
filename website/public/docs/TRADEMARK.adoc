= CentrMark trademark and naming guidelines

These guidelines complement the link:LICENSE[GNU AGPL-3.0] license for this repository. They are not a substitute for legal advice.

== Purpose

* Clarify who may call an implementation or document "CentrMark" in ways that imply official status.
* Encourage compatible implementations while discouraging incompatible variants that reuse the project name in misleading ways.

== Permitted use (no permission needed)

You may truthfully describe products or documents as:

* _CentrMark-compatible_ or _supports CentrMark syntax_, if they substantially follow the published grammar and semantics in this repository or its documented successors.
* Referencing the `.cmk` extension when discussing file types you handle.

Always provide accurate attribution and, where reasonable, link to this project so users know where the canonical specification lives.

== Certification wording

You may use "Certified CentrMark" (and similar certified-level wording) only if:

* the implementation appears in the public certification registry for the relevant spec baseline/version, and
* you do not claim a certification level beyond the registry entry.

If you have not been verified, prefer descriptive phrasing like "CentrMark-compatible" instead.

== Restricted use

Do **not**:

* Imply endorsement, sponsorship, or official partnership with the CentrMark maintainers without their explicit written permission.
* Name a materially incompatible markup dialect _CentrMark_, _CMK_, or market it under names readily confused with this project’s identifiers.

Forks that intentionally diverge from the canonical specification should choose a **distinct project name** and clearly describe how they differ. That protects users and reduces gratuitous format fragmentation.

== Canonical specification

The authoritative specification and grammar for this format are maintained in this repository (and successors designated by the maintainers). Others should treat improvements here as the reference track for what "CentrMark" describes when used without qualification.
